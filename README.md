# sensor-monitoring-station

honestly this started because i wanted to prove to myself that i actually understand what happens inside a chemical reactor and not just theoritically. a CSTR or a continuous stirred tank reactor is one of the first things they teach you in process engineering and i always wondered if i could simulate the whole thing in code. so i did. the simulator generates live sensor data for temperature, concentration, pressure and flow rate just like a real reactor would, with noise and everything. the problem with real sensors is the readings are always jumping around so i built a kalman filter that cleans up the signal and gives you the actual value instead of the noisy one.
then i added a PID controller which is basically what keeps the reactor at the temperature you want. you set a target, the controller figures out how much coolant to push through and keeps adjusting every fraction of a second. the part that took me the longest was the fault detection, i used something called a z-score which looks at how far a reading is from the normal baseline and if it goes too far it fires an alarm. you can test it yourself by clicking the inject fault buttons and watching the dashboard go red. theres also a csv export button now so you can download all the data and analyse it seperately. building this made alot of concepts click for me that just reading about them never did, things like why kalman filters are used in instrumentation, how PID tuning actually effects the response, and what fault detection looks like in a real control system.
